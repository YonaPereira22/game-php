# Hitos de Base de Datos

Fecha: 2026-05-05
Proyecto: game-php

## Objetivo

Estandarizar la gestion de la base de datos mediante scripts SQL versionados con nomenclatura `NN_descripcion.sql`, tomando como referencia los datos reales disponibles en `scripts/data`.

## Hitos implementados

### 1) Estructura base versionada

Se creo la carpeta `db/migrations` y se agrego el script:

- `01_crear_estructura_base.sql`

Este script:
- Crea la base `game` con `utf8mb4`.
- Crea las tablas iniciales `games`, `users` y `votes`.
- Define claves primarias, foranea de votos y restricciones unicas base.

### 2) Migracion de usuarios para OAuth y auditoria

Se agrego el script:

- `02_modificar_users_oauth.sql`

Este script:
- Agrega campos `email`, `google_id`, `google_picture`, `created_at`, `last_login`.
- Cambia `password` para permitir `NULL`.
- Permite `email` nullable para coexistencia entre login local y Google OAuth.
- Crea indices unicos para `email` y `google_id`.

### 3) Migracion de juegos para enlace de publicacion

Se agrego el script:

- `03_modificar_games_github_link.sql`

Este script:
- Agrega la columna `github_link` en `games`.
- Agrega indice unico para evitar duplicados de enlace.

### 4) Carga inicial de catalogo de juegos

Se agrego el script:

- `04_cargar_datos_games.sql`

Este script:
- Inserta los juegos de referencia provenientes de `scripts/data/games.sql`.
- Usa `ON DUPLICATE KEY UPDATE` para permitir rejecucion sin duplicar filas.

### 5) Carga inicial de usuarios

Se agrego el script:

- `05_cargar_datos_users.sql`

Este script:
- Inserta usuarios de referencia de `scripts/data/users.sql`.
- Conserva hashes y datos OAuth.
- Usa `ON DUPLICATE KEY UPDATE` para sincronizacion idempotente.

### 6) Carga inicial de votos

Se agrego el script:

- `06_cargar_datos_votes.sql`

Este script:
- Inserta votos de referencia de `scripts/data/votes.sql`.
- Usa `ON DUPLICATE KEY UPDATE` para permitir actualizacion de valoracion y fecha al rejecutar.

## Orden recomendado de ejecucion

Nota importante para hosting compartido:
- Estos scripts no crean ni seleccionan base de datos automaticamente.
- Debes ejecutarlos dentro de tu base asignada por el proveedor (por ejemplo `u952965051_game`).

1. `db/migrations/01_crear_estructura_base.sql`
2. `db/migrations/02_modificar_users_oauth.sql`
3. `db/migrations/03_modificar_games_github_link.sql`
4. `db/migrations/04_cargar_datos_games.sql`
5. `db/migrations/05_cargar_datos_users.sql`
6. `db/migrations/06_cargar_datos_votes.sql`

## Resultado

Con estos hitos, la base queda:
- Versionada por pasos.
- Compatible con login local y Google OAuth.
- Preparada para catalogo de juegos con enlace externo opcional.
- Cargada con los datos de referencia del proyecto.
