🌐 Viaje de un dato por Internet

Simulá cómo viaja una solicitud web desde tu PC hasta el servidor

💻
1. Tu computadora
Escribís una URL. El navegador crea un paquete de datos con esa solicitud.
📦 GET /index.html HTTP/1.1
📡
2. Router doméstico
Tu router asigna una dirección IP y reenvía el paquete hacia Internet por el proveedor de servicio (ISP).
🔢 IP: 192.168.1.x → ISP
🏢
3. ISP – Proveedor de Internet
El ISP conecta tu paquete con la red troncal de Internet, la "columna vertebral" global.
🌐 DNS lookup → IP del servidor
🔀
4. Routers de Internet
Múltiples routers deciden el mejor camino. El protocolo IP guía cada paquete salto a salto.
⚡ ~15 hops hasta el destino
🖥️
5. Servidor web
El servidor recibe la solicitud, procesa y envía la respuesta (HTTP 200 OK) con el contenido pedido.
✅ HTTP/1.1 200 OK + HTML
🖼️
6. Tu navegador
El navegador recibe los paquetes, los reensambla y renderiza la página que ves en pantalla.
🎉 ¡Página cargada!
HTTP/S
HyperText Transfer Protocol
Protocolo para transferir páginas web. La "S" agrega encriptación SSL/TLS.
DNS
Domain Name System
Traduce nombres como "google.com" en direcciones IP numéricas.
IP
Internet Protocol
Divide datos en paquetes y los dirige a través de la red usando direcciones únicas.
TCP
Transmission Control Protocol
Garantiza que todos los paquetes lleguen completos y en orden.
